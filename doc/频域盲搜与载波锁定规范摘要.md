# 5G NR 频域盲搜与载波锁定规范摘要

本文档从 3GPP TS 38.101-1 和 TS 38.211 规范中提取，涵盖小区搜索过程中频域盲搜与载波锁定的关键步骤、规范条款及参数定义。

---

## 1. 小区搜索流程概述

UE 初始接入时执行小区搜索（Cell Search），主要步骤：

1. **频域盲搜 PSS**：在同步栅格（Synchronization Raster）候选频点上检测 PSS
2. **时域同步**：通过 PSS 相关检测获取符号定时
3. **频偏估计与补偿**：估计并补偿载波频偏
4. **SSS 检测**：在 PSS 确定的位置检测 SSS，获取完整物理层小区 ID
5. **PBCH 解调**：解调 PBCH 获取 MIB 和系统信息

---

## 2. 同步栅格（Synchronization Raster）

### 2.1 规范来源

**3GPP TS 38.101-1 Clause 5.4.3**

> The synchronization raster indicates the frequency positions of the synchronization block that can be used by the UE for system acquisition when explicit signalling of the synchronization block position is not present.

### 2.2 GSCN 定义

全局同步栅格用 GSCN（Global Synchronization Channel Number）编号，对应 SS Block 中心频率 SSREF。

**Table 5.4.3.1-1: GSCN 参数（> 3 MHz 带宽）**

| 频率范围 | SS Block 频率位置 SSREF | GSCN | GSCN 范围 |
|---------|------------------------|------|----------|
| 0 – 3000 MHz | N × 1200 kHz + M × 50 kHz<br>N = 1:2499, M ∈ {1, 3, 5} | 3N + (M-3)/2 | 22 – 7498 |
| 3000 – 24250 MHz | 3000 MHz + N × 1.44 MHz<br>N = 0:14756 | 7499 + N | 7499 – 22255 |
| 24250 – 100000 MHz | 24250.08 MHz + N × 17.28 MHz<br>N = 0:4383 | 22256 + N | 22256 – 26639 |

### 2.3 GSCN 到频率转换实现

```python
def gscn_to_frequency(gscn: int) -> float:
    if gscn <= 7498:
        # 0 - 3000 MHz
        n = (gscn + 1) // 3
        m_remainder = (gscn + 1) % 3
        m = {0: 1, 1: 3, 2: 5}[m_remainder]
        return n * 1.2e6 + m * 50e3
    elif gscn <= 22255:
        # 3000 - 24250 MHz
        n = gscn - 7499
        return 3000e6 + n * 1.44e6
    else:
        # > 24250 MHz
        n = gscn - 22256
        return 24250.08e6 + n * 17.28e6
```

### 2.4 各频段有效 GSCN 范围

**Table 5.4.3.3-1: 各工作频段同步栅格条目（部分）**

| NR 频段 | SS Block SCS | SS Block Pattern | GSCN 范围 (First – <Step> – Last) |
|--------|-------------|-----------------|----------------------------------|
| n1 | 15 kHz | Case A | 5279 – <1> – 5419 |
| n41 | 15 kHz | Case A | 6246 – <3> – 6717 |
| n41 | 30 kHz | Case C | 6252 – <3> – 6714 |
| n78 | 30 kHz | Case C | 7711 – <1> – 8051 |
| n104 | 30 kHz | Case C | 9882 – <7> – 10358 |

**SS Block Pattern 说明**（定义于 TS 38.213 Clause 4.1）：
- **Case A**：SCS = 15 kHz
- **Case B**：SCS = 30 kHz
- **Case C**：SCS = 30 kHz（FR1）
- **Case D**：SCS = 120 kHz（FR2）
- **Case E**：SCS = 240 kHz（FR2）

## 3. PSS（主同步信号）

### 3.1 规范来源

**3GPP TS 38.211 Clause 7.4.2.2**

### 3.2 物理层小区 ID

共有 1008 个唯一物理层小区 ID：

```
N_ID^cell = 3 × N_ID^1 + N_ID^2

其中：
- N_ID^1 ∈ {0, 1, ..., 335}  (SSS 检测得到)
- N_ID^2 ∈ {0, 1, 2}         (PSS 检测得到，对应 3 个扇区)
```

### 3.3 PSS 序列生成

**Clause 7.4.2.2.1 Sequence generation**

PSS 基于长度 127 的 m 序列，生成多项式：x(i+7) = x(i+4) + x(i) mod 2

```
d_PSS(n) = 1 - 2 × x[(n + 43 × N_ID^2) mod 127]

其中：
- n = 0, 1, ..., 126
- x(i) 为 m 序列，初始状态：[x(6), x(5), ..., x(0)] = [1, 1, 1, 0, 1, 1, 0]
```

### 3.4 PSS 映射到物理资源

**Clause 7.4.3.1.1 Mapping of PSS within an SS/PBCH block**

PSS 映射到 SS/PBCH block 的第 0 个 OFDM 符号，占用子载波 56 ~ 182（共 127 个子载波）。

**Table 7.4.3.1-1: SS/PBCH block 资源映射**

| 信道/信号 | OFDM 符号号 l | 子载波号 k |
|----------|--------------|-----------|
| PSS | 0 | 56, 57, …, 182 |
| SSS | 2 | 56, 57, …, 182 |
| PBCH | 1, 3 | 0, 1, …, 239 |
| PBCH | 2 | 0, 1, …, 47, 192, 193, …, 239 |

## 4. SS/PBCH Block 结构

### 4.1 规范来源

**3GPP TS 38.211 Clause 7.4.3.1**

### 4.2 时频结构

- **时域**：4 个 OFDM 符号（编号 0 ~ 3）
- **频域**：240 个连续子载波（编号 0 ~ 239）

```
符号 0: PSS
符号 1: PBCH + DM-RS
符号 2: SSS + PBCH(部分) + DM-RS
符号 3: PBCH + DM-RS
```

### 4.3 SS/PBCH Block 时域位置

**Clause 7.4.3.2**：时域位置由 TS 38.213 Clause 4.1 定义，取决于 SS Block Pattern（Case A/B/C/D/E）。

## 5. 频域盲搜实现要点

### 5.1 搜索空间构建

1. 根据目标频段确定 GSCN 范围（First, Last, Step）
2. 将 GSCN 转换为绝对频率
3. 对每个候选频点生成频偏补偿后的接收信号

### 5.3 检测输出

PSS 检测成功后获得：
- **Timing Offset**：符号定时偏移
- **N_ID^2**：扇区 ID（0, 1, 2）
- **GSCN**：SS Block 中心频率对应的 GSCN
- **Frequency Offset**：载波频偏估计
